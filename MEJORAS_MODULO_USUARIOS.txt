PARTE 1 — FALENCIAS FUNCIONALES (bugs y lógica rota)
================================================================================

[F1] NuevoTramiteForm.js — DESCRIPCIÓN LIMITADA A 50 CARACTERES (crítico)
  - Línea 444: maxLength={50} y línea 169 valida descripcion.length > 50
  - Un campo de descripción de un trámite ambiental con 50 chars es inutilizable
  - Un permiso forestal necesita al menos 200-500 chars para describir el proyecto
  SOLUCIÓN: Cambiar maxLength a 500 y validar minLength >= 30 para asegurar
  que el ciudadano escriba algo útil, no solo "hola".

[F2] NuevoTramiteForm.js — RESETEO INCOMPLETO AL CERRAR (bug)
  - handleGuardar() resetea formData pero NO resetea municipiosDisponibles
  - Si el usuario abre el form de nuevo, el select de municipios queda populado
  - Tampoco hay confirmación si el usuario cierra con datos a medio llenar
  SOLUCIÓN:
    const resetForm = () => {
      setFormData({ tipo: TIPOS_TRAMITE.PERMISO, subtipo: '', ... });
      setDocTipo('V');
      setMunicipiosDisponibles([]);
      setError(null);
    };
  Usar resetForm() en onClose y en onSave. Agregar confirmación de cierre
  si hay datos (como hace DenunciaForm con useConfirmModal).

[F3] NuevoTramiteForm.js — VALIDACIÓN DE EMAIL NO BLOQUEA EN TIEMPO REAL
  - email es opcional pero si se escribe un email inválido, el error solo
    aparece al submit. El usuario no sabe que está mal mientras escribe.
  SOLUCIÓN: Agregar validación onBlur en el input de email mostrando
  error inline igual que el campo cedulaRif que ya lo hace bien.

[F4] MisTramites/index.js — FILTROS PILL SIN IMPLEMENTAR (UX rota)
  - El index.js planificaba "filtros rápidos tipo pills" pero la implementación
    delega todo a MisTramitesTabla con un CFormSelect genérico (dropdown)
  - El CFormSelect aparece aunque tramitesFiltrados.length === 0 queda oculto
    porque el empty state retorna antes del render del select
  SOLUCIÓN: Mover los filtros pill al index.js como se planeó, encima de la
  CCard de la tabla, usando CBadge/CButton pills igual que en denuncias.js

[F5] MisDenuncias/index.js — SOLO ES UN RE-EXPORT (stub incompleto)
  - El archivo es: export { default } from '../denuncias/denuncias';
  - Por lo tanto MisDenuncias usa useDenuncias (MOCK_DATA volátil en memoria)
    en lugar del nuevo useMisDenuncias que sí usa gestionService
  - El hook useMisDenuncias.js existe (3316 bytes) pero NUNCA SE USA
  SOLUCIÓN: Implementar MisDenuncias/index.js como vista real que use
  useMisDenuncias y reutilice los componentes de denuncias/components/

[F6] useMisDenuncias.js — VERIFICAR QUE gestionService EXPORTA lo necesario
  - Si gestionService no tiene crearDenuncia() o la firma es diferente,
    el hook fallará silenciosamente. Revisar el servicio antes de usar.

[F7] perfilUsers.js — DATOS MOCK NO PERSISTENTES
  - formData se inicializa desde MOCK_USUARIO (constante hardcodeada)
  - guardarCambios() actualiza el estado local pero NO guarda en localStorage
  - Al refrescar la página, los cambios se pierden
  SOLUCIÓN mínima: guardar en localStorage al guardar:
    localStorage.setItem('perfilCiudadano', JSON.stringify(sanitizedData));
  Y cargar al montar:
    const saved = localStorage.getItem('perfilCiudadano');
    setFormData(saved ? JSON.parse(saved) : MOCK_USUARIO);

[F8] denunciasTable.js — LÍNEA 546 TIENE CÓDIGO COMENTADO MEZCLADO CON JSX
  - Línea 546: "// En DenunciasTable.jsx, en la sección de ubicación..."
  - Esto es un comentario de instrucción del agente que quedó en el código
  - Aunque es un comentario JS válido, ensucia el código y puede confundir
  SOLUCIÓN: Eliminar ese bloque de comentario (líneas 546-549 aprox.)

[F9] denunciasForm.js — TÍTULO Y DESCRIPCIÓN SE GENERAN AUTOMÁTICAMENTE
  - La descripción enviada al admin es: "Incidente de tipo X en Y, Estado"
  - El ciudadano no puede escribir su propia descripción del incidente
  - Esto hace que todas las denuncias del mismo tipo/zona sean idénticas
  SOLUCIÓN: Agregar un campo CFormTextarea en el paso 1 donde el ciudadano
  describa con sus palabras qué ocurrió (min 30 chars, requerido).
  El titulo puede seguir siendo auto-generado.
